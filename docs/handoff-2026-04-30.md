## What Was Done

### 1. Forensic Analysis of Specific Failures

Investigated these recurring failures on smollm2-360m:

| Source | Bad Output | Root Cause |
|--------|-----------|------------|
| "You have a nice house" | "Te has dado una buena casa" | Model maps possessive "have" to auxiliary verb *haber* ("has") instead of *tener* ("tienes"). |
| "I need a bus ticket" | "Usted necesita un campero..." | Model hallucinates unrelated nouns when it doesn't know the dialect term. |
| "I took the elevator to the 5th floor" | "Me pasé el elevator al 5ª escena" | Leaves English words untranslated; substitutes wrong Spanish words ("escena" = scene). |
| "A strawberry is sweet" | "Un estrella es muy amarillo" | Confuses "strawberry" with "star" (*estrella*) — orthographic confusion in weak model weights. |
| "You do it well" | "¡Bienvenido!" / "Yo lo hago bien" | Model treats "You..." input as being addressed TO the assistant; responds conversationally or flips to first person. |

### 2. Fixes Applied

#### a) Voseo Adapter — `haber` False Positive (packages/types/src/verb-conjugations.ts)
- **Problem:** `haber` conjugation had `present_2s: "has"` for tú and `"habés"` for vos. When the model mistranslated "You have" as "Te has dado", the voseo adapter swapped "has" → "habés", compounding the error to "Te habés dado".
- **Fix:** Changed vos form to `"has"` (same as tú) because `"habés"` does not exist in standard voseo. Added comment explaining why.

#### b) Garbage Detection Expansion (packages/providers/src/providers/llm.ts)
- Added patterns for:
  - Standalone "¡Bienvenido!" (conversational response)
  - "Entiendo. Estoy listo..." (Spanish preamble fragments)
  - "frutilla...frutilla" repetition (nonsense)
  - "campero" (known hallucination for bus)
- Added **length-ratio check**: output >4× or <15% of source length triggers garbage retry.

#### c) Preamble Stripping — Spanish Patterns (packages/providers/src/providers/llm.ts)
- Added Spanish conversational preambles to `PREAMBLE_PATTERNS`:
  - `¡Bienvenido!`
  - `Entiendo. Estoy listo...`
  - `Claro, puedo...`

#### d) Retry Prompt Preserves Vocab Hints (packages/providers/src/providers/llm.ts)
- **Problem:** When garbage was detected, the retry used `buildStrictSystemPrompt()` which was a generic prompt with **no dialect vocabulary hints**. The retry was therefore *less informed* than the original call.
- **Fix:** `buildStrictSystemPrompt()` now accepts optional `dialect` and `text` parameters and includes targeted vocab hints in the retry prompt.

#### e) `fixUntranslatedWords` Post-Processor (packages/providers/src/providers/llm.ts)
- **New function:** Scans output for English words that match dictionary concept names (e.g., "elevator", "strawberry", "bus") and replaces them with the dialect-preferred Spanish term.
- **Placement:** Runs after lexical substitution, before voseo adapter.
- **Safety:** Only matches exact concept names (not gloss words) to avoid false positives.
- **Example:** `"Me pasé el elevator al 5°."` → `"Me pasé el elevador al 5°."` (es-MX)

#### f) System Prompt Refinement
- Reverted an experimental numbered-rules format that made weak models perform *worse* (dropped from 64% to 56%).
- Kept the cleaner prompt: "You are a Spanish translation engine. Translate the input text to Spanish... Preserve grammatical person..."
- Added `buildTargetedGrammarHint()` function (in code but not actively used in the final prompt, after A/B testing showed it hurt weak-model performance).

## Architecture for the Next Person

### Key Files

| File | Purpose |
|------|---------|
| `packages/providers/src/providers/llm.ts` | LLM provider: prompts, garbage detection, retry logic, post-processing pipeline |
| `packages/providers/src/lexical-substitution.ts` | Deterministic vocabulary swap (avoid → preferred per dialect) |
| `packages/providers/src/voseo-adapter.ts` | tú → vos `present_2s` swapper |
| `packages/types/src/dialectal-dictionary.json` | Master dictionary of ~834 dialect vocabulary entries |
| `packages/types/src/dialectal-dictionary.ts` | Types and runtime loader for the dictionary |
| `packages/types/src/verb-conjugations.ts` | Verb conjugation tables for voseo |
| `scripts/real-benchmark.mjs` | End-to-end benchmark using actual provider pipeline |

### Post-Processing Pipeline Order (in `_runPostProcessing`)

```
1. applyLexicalSubstitution     — wrong-dialect → right-dialect words
2. fixUntranslatedWords          — English concept names → Spanish (NEW)
3. applyVoseo                    — tú → vos for voseo dialects
4. applyAgreementFixes           — gender/number agreement
5. normalizePunctuation          — smart quotes, em-dashes, etc.
6. fixAccentuation               — missing tildes
7. normalizeCapitalization       — sentence case
8. normalizeTypography           — spacing, ellipsis
9. restoreSentinels              — placeholder tokens (ALWAYS LAST)
```

### Prompt Architecture for Compact Models

For models detected as `isCompactModel()` (≤8B, MiniMax, or `/\d+m\b/` million-param models):
- **System prompt** contains ALL instructions + targeted vocabulary hints
- **User prompt** contains ONLY the source text — weak models cannot separate instructions from content

### Garbage Detection Flow

```
Call 1 → output
  ↓
isGarbageOutput(source, output)?
  ↓ yes
Retry with buildStrictSystemPrompt(dialect, text)  ← now preserves hints
  ↓
isGarbageOutput(source, output)?
  ↓ yes
Throw "LLM produced garbage output after retry"
```

## Open Issues / Next Steps

1. **"You do it well" → "Yo lo hago bien" (perspective change)**
   - The model treats "You..." as being addressed to the assistant. This is instruction-tuning behavior.
   - Potential fix: Add a post-processor that detects "Yo" when source starts with "You" and flips to appropriate tú/vos form. **Risk:** Could break legitimate first-person translations.

2. **"You have a nice house" → "Te has dado" (haber vs tener)**
   - The 360M model's weights associate "have" with "haber" in this context.
   - Potential fix: Add a post-processor that detects "Te has [noun phrase]" when source is "You have a [noun]" and replaces with "Tienes [noun phrase]". **Risk:** Very specific heuristic; could break auxiliary uses.

3. **Strawberry → star / fruits**
   - Weak model confuses orthographically/phonetically similar words.
   - Potential fix: Add semantic validation layer (e.g., embedding similarity) to flag outputs that are semantically unrelated to source. **Cost:** Adds latency and complexity.

4. **Bus ticket hallucinations**
   - Model invents wrong nouns. Vocab hints don't help because the model ignores them.
   - Potential fix: Two-call pipeline (translate → dialect-adapt) might give weak models a simpler task per call. Currently disabled by default for compact models.

5. **Model capacity threshold**
   - Data shows a sharp threshold: models ≥4B get 100%, ~600M gets 87%, ~350M gets 56-69%.
   - If 100% reliability is required on these test cases, the systemic fix is to use a larger model (qwen3.5-2B or above).
